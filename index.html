С помощью PuTTY заходим на сервер

С помощью команды ping на учебном сервере узнали IP-адрес веб-сервера kubsu.ru. Сама по себе команда ping необходимя для того, чтобы проверить подключение к другому компьютеру на уровне IP

С помощью команды nslookup узнали A-записи и MX-записи домена kubsu.ru и kubsu-dev.ru. Запись А нужна для связи домена с IP адресом сервера, запись МХ отвечает за сервер, через который будет работать почта

С помощью команды whois узнали дату регистрации домена kubsu.ru

С помощью команды whois узнали дату регистрации домена kubsu-dev.ru

Далее генерируем ssh ключи с помочью команды ssh keygen. В процессе генерируется два ключа: открытый и закрытый. Для того, чтобы добавить необходимые файлы на сервер, нам нужен открытый ключ

Заходим на GitHub, создаем репозиторий, в который загружаем все необходимые файлы для сайта. Далее, в настройках заходим в раздел Deploy key и добавляем сгенерированный ранее ключ.

После того, как мы добавили открытый ssh ключ в настройках у нашего репозитория кроме обычной http ссылки появилась также ssh ссылка. Это нам понадобится в дальнейшем, чтобы соедениться с репозиторием и клонировать его на сервер

Так как ранее мы подключились к серверу, мы имеем доступ к его каталогам. По заданию необходимо все файлы добавить в каталог www. Для этого с помощью команды cd www мы заходим в этот каталог. Следующая команда git clone клонирует репозиторий в эту папку, обязательно указываем ssh, который мы добавили ранее

С помощью FileZilla мы устанавливаем sftp соединение (протокол безопасной передачи файлов), вводим логин и пароль от сервера. Можно увидеть, что в каталоге www появилась папка lab1 - это название моего репозитория, в этой папке находятся все файлы, лежащие на GitHub

Также, по заданию необходимо было скопировать файлы на локальный компьютер из каталога www, что и иллбстрирует данный скриншот
